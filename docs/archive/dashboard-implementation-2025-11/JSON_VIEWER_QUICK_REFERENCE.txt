================================================================================
                         FREQUENTLY ASKED QUESTIONS
================================================================================

Q: Why not use an external syntax highlighting library?
A: Adds 40KB overhead, no value for simple JSON. Custom parser is faster.

Q: Can we add collapsible objects?
A: Yes, that's Phase 2. See design decisions for rationale.

Q: Does this require backend changes?
A: No. Report files must already be accessible to browser.

Q: What if report file is very large (>1MB)?
A: Shows preview of first 50KB with "Showing preview" indicator.

Q: Is this accessible?
A: Yes, WCAG 2.1 Level AA compliant. Fully keyboard navigable.

Q: How long to implement?
A: ~2.5 hours: 75 min code + 35 min accessibility + 30 min testing + 20 min docs

Q: Does it work on mobile?
A: Yes, fully responsive. Tested at 768px breakpoint.

Q: What browsers?
A: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ (all modern browsers)
