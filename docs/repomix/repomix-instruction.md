# Repomix Repository Analysis Instructions

This repository bundle is generated by Repomix for AI-assisted code review and architecture analysis.

## Bundle characteristics

- Output format: XML
- Compression enabled
- Comments and empty lines removed
- Directory structure and summaries included for navigation
- Docs are intentionally excluded to reduce token bloat
- Git diffs excluded; git logs disabled/minimized

Treat this bundle as read-only context. Prefer minimal, targeted fixes.

## What to focus on first

Prioritize executable/runtime code and entrypoints:

1. `api/` — backend routes, middleware, websockets, schemas, API utilities
2. `sidequest/` — pipelines, orchestrators, scanners, workers, report generation
3. `frontend/` — UI components, websocket hooks, API client, state/store
4. `scripts/` — automation and operational tooling
5. `config/` — runtime and scanning configuration

## Navigation strategy

1. Start with directory structure + top files list.
2. Identify entrypoints (servers, orchestrators, workers).
3. Follow imports/dependency roots outward.
4. Inspect error-handling, retries/timeouts, and async coordination.
5. Keep recommendations small, localized, and concrete.

## What to ignore

- Large documentation trees / archives
- Generated artifacts (including previous repomix outputs)
- Media/binaries and large datasets
